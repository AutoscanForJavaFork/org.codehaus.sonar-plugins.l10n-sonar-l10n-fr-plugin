Lorsqu'un bloc de code est vide, il s'agit la plupart du temps d'un oubli : il manque réellement du code source à cet endroit. Ainsi, un bloc vide doit soit être rempli, soit être supprimé. A noter qu'un bloc contenant un commentaire n'est pas considéré comme vide.

Le code suivant illustre cette règle:

void doSomething() {
  int myVar = 0;
  for (int i = 0; i< 4; i++) // Non-Compliant
  {
  }
  for (int i = 0; i< 4; i++); // Compliant (il n'y a aucun bloc)

  if (myVar == 4) // Compliant (le bloc contient un commentaire)
  {
    // nothing to do
  }
  else // Compliant
  {
    doSomething();
  }

  try // Non-Compliant
  {
  }
  catch (Exception e) // Non-Compliant
  {
  }
}
