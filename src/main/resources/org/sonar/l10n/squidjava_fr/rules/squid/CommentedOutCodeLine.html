Voici les raisons pour lesquelles le code commenté pose problème :

  • Souvent, il lève plus de question qu'il n'offre de réponde
  • Tout le monde oubliera rapidement si le code commenté est pertinent ou pas
  • Le code commenté distrait l'attention lors de la lecture du code car il stoppe le flux de lecture
  • C'est un mauvais moyen d'émuler un outil de gestion de configuration : Subversion, CVS ou Git sont bien plus efficaces !
  • Le fait fait de comprendre pourquoi un code a été commenté peut inutilement prendre un certain temps
